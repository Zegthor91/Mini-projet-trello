# Mini Trello Project(Alpine.js)

## Contexte

L'objectif de ce projet est de concevoir un mini trello : type *application web de gestion de tâches*.

---

## Objectifs pédagogiques

- Manipuler les données réactives avec **Alpine.js** (`x-data`, `x-model`, `x-for`, `x-show`, etc.)  
- Gérer les événements utilisateurs (`@click`, `@submit`, etc.)  
- Mettre en place la **persistance locale** avec `localStorage`  
- Structurer un projet web complet (HTML + Tailwind + Alpine.js)  
- Concevoir une interface **ergonomique, claire et responsive**

## Cahier des charges

### Fonctionnalités obligatoires (50% de la note)

- Affichage des tâches sous forme de cartes (titre, description, statut)  
- Ajout d’une tâche via un formulaire (statut par défaut = “À faire”)  
- Modification d’une tâche (titre, description ou statut)  
- Suppression d’une tâche  
- Changement du statut d’une tâche via un menu ou bouton  
- Sauvegarde des tâches dans `localStorage` (persistance après rechargement)

---

### Fonctionnalités avancées (30% de la note)

- Filtrer les tâches par statut  
- Rechercher une tâche par titre  
- Afficher le nombre total de tâches par statut  
-  Ajouter une confirmation avant la suppression

---

### Présentation et UX (20% de la note)

- Interface claire et lisible  
- Design responsive (mobile et desktop)  
- Boutons intuitifs et labels explicites  
- Navigation fluide et esthétique
